데이터를 넣고, 보여주고,
바꾸는 법

왼쪽은 코드, 오른쪽은 그 코드가 실제로 돌아가는 화면입니다. 버튼을 누르고 글자를 입력하면 데이터가 어떻게 따라 움직이는지 눈으로 확인할 수 있습니다.

●  오른쪽 영역은 진짜 Vue가 구동 중입니다 — 마음껏 눌러보세요
⚠ Vue 라이브러리를 불러오지 못해 오른쪽 데모가 동작하지 않을 수 있습니다(네트워크 환경 영향). 코드 설명은 그대로 보실 수 있습니다.
{{ }}

화면에 글자로 표시

데이터를 텍스트로 꺼내 보여줍니다.

{{ name }} → 홍길동
:

속성 값에 연결 (v-bind)

데이터를 HTML 속성에 넣습니다.

:key="id" · :src="url"
@

동작에 반응 (v-on)

클릭·입력 등 이벤트를 듣습니다.

@click · @input · @keyup
⚠

자주 하는 오해. @key 라는 건 없습니다. 목록에서 쓰는 건 :key(데이터를 key 속성에 연결). @는 오직 이벤트를 들을 때만 씁니다.

data + {{ }} + @click

① 넣기 → 보여주기 → 바꾸기 (데이터 한 바퀴)

data에 count를 넣고, {{ count }}로 보여주고, 버튼 클릭(@click)으로 this.count 를 바꿉니다.

<script>
export default {
  data() {
    return { count: 0 };   // ① 넣기
  },
  methods: {
    plus() { this.count++; }   // ③ this 로 바꾸기
  }
};
<template>
<p>{{ count }}</p>             <!-- ② 보여주기 -->
<button @click="plus">+1</button>
실제 동작
{{ count }}

버튼을 누르면 count 데이터가 바뀌고, {{ count }} 부분이 자동으로 다시 그려집니다.

v-model

② 입력창과 데이터를 양쪽으로 묶기

v-model 한 줄이면 입력한 글자가 곧바로 name 데이터가 되고, 그 데이터가 화면에도 즉시 반영됩니다.

<script>
data() {
  return { name: "" };
}
<template>
<input v-model="name">
<p>{{ name }}님, 환영합니다</p>
실제 동작
{{ name || '...' }}님, 환영합니다

타이핑하는 즉시 아래 글자가 따라옵니다. 이게 "양방향 바인딩"입니다.

v-if / v-else

③ 조건에 따라 화면을 넣었다 뺐다

show가 참이면 v-if 쪽이, 거짓이면 v-else 쪽이 그려집니다. 거짓인 요소는 화면에서 아예 사라집니다.

<script>
data() {
  return { show: true };
}
<template>
<button @click="show = !show">토글</button>

<p v-if="show">보입니다 👀</p>
<p v-else>숨었습니다 🙈</p>
실제 동작
show = {{ show }}
보입니다 👀 숨었습니다 🙈

!show는 "반대로 뒤집기"입니다(참↔거짓). v-if가 거짓이면 그 문장은 DOM에서 통째로 빠집니다.

v-for + :key

④ 목록(배열)을 반복해서 그리기

배열을 v-for로 한 줄씩 그립니다. 이때 각 줄을 구분할 고유값을 :key에 연결해 줍니다(@key 아님!).

<script>
data() {
  return { todos: [ /* {id, text} 배열 */ ] };
},
methods: {
  remove(id) {
    this.todos = this.todos.filter(t => t.id !== id);
  }
}
<template>
<li v-for="t in todos" :key="t.id">
  {{ t.text }}
  <button @click="remove(t.id)">삭제</button>
</li>
실제 동작
  • {{ t.text }}
목록이 비었습니다. 위에서 추가해 보세요.

추가하면 배열에 들어가 한 줄이 늘고, 삭제하면 그 줄만 빠집니다. @keyup.enter로 엔터 입력도 받았습니다.

: (v-bind)

⑤ 데이터를 "속성 값"에 넣기

글자 표시가 아니라 style·disabled 같은 속성에 데이터를 꽂을 땐 콜론(:)을 씁니다.

<script>
data() {
  return { size: 20 };
}
<template>
<input type="range" v-model="size">

<p :style="{ fontSize: size + 'px' }">
  글자 크기 {{ size }}px
</p>
실제 동작
글자 크기 {{ size }}px

슬라이더(size 데이터)를 움직이면 :style에 연결된 글자 크기가 실시간으로 바뀝니다.

전부 합쳐보기

⑥ 미니 장바구니 — 배운 걸 한 화면에

v-model(입력) · @click(담기) · v-for·:key(목록) · {{ }}(합계) · v-if(빈 목록 안내) · :disabled(빈 입력 막기)를 모두 씁니다.

<script>
data() {
  return { item: "", price: 0, cart: [] };
},
computed: {
  total() {                       // 합계 자동 계산
    return this.cart.reduce((s, c) => s + c.price, 0);
  }
},
methods: {
  add() {
    if (!this.item) return;
    this.cart.push({ id: Date.now(), name: this.item, price: +this.price });
    this.item = ""; this.price = 0;
  }
}
실제 동작
  • {{ c.name }} {{ c.price.toLocaleString() }}원
담은 상품이 없습니다.
합계 {{ total.toLocaleString() }}원 ({{ cart.length }}개)
작성 방식 2가지

⑦ setup은 뭐고, 나는 뭘 써야 하나?

Vue 코드를 쓰는 방식은 두 가지입니다. 같은 카운터를 양쪽으로 써봤습니다. 결과는 똑같습니다.

Options API · 이 프로젝트 ✓
export default {
  data() {
    return { count: 0 };  // 데이터 칸
  },
  methods: {
    plus() { this.count++; }  // 함수 칸
  }
};
Composition API · setup (Vue 3)
import { ref } from "vue";
export default {
  setup() {
    const count = ref(0);        // 데이터
    const plus = () => count.value++; // 함수
    return { count, plus };       // 화면에 넘김
  }
};

차이. Options API는 data·methods 같은 정해진 칸에 나눠 담습니다. Composition API는 setup() 함수 하나에 다 모으고 값을 ref()로 감쌉니다(읽을 땐 .value).

결론. 이 프로젝트는 Vue 2 + Options API입니다. setup은 Vue 3에서 주로 쓰는 방식이라 여기서는 쓸 일이 없습니다. data·methods·computed·mounted만 익히면 충분합니다. setup 코드를 마주치면 "아, Vue 3 방식이구나" 정도로 알아두시면 됩니다.
Options API이걸 쓰세요   setup이 프로젝트에선 불필요

생명주기 · created / mounted

⑧ 화면이 켜질 때 자동으로 실행되는 칸

컴포넌트는 정해진 순서로 "태어납니다". 그 특정 시점에 자동 실행되는 칸이 생명주기 훅입니다. 가장 많이 쓰는 건 mounted입니다.

<script>
export default {
  data() {
    return { list: [] };
  },
  created() {
    // 데이터 준비됨 (화면 그리기 전)
    // 보통 여기서 서버 호출 시작
  },
  mounted() {
    // 화면이 실제로 붙은 직후
    // DOM을 만져야 할 때 사용
  }
};
이 데모가 켜질 때 실제로 실행된 순서
data()
데이터 초기값을 준비
컴포넌트 생성 시작
created()
데이터 준비 완료 · 아직 화면은 없음
{{ logTime('created') }}
화면 렌더링
template → 실제 DOM으로 그려짐
mounted()
화면이 붙음 · 이제 DOM 조작 가능
{{ logTime('mounted') }}

created vs mounted — 서버에서 데이터만 가져오면 created, 그려진 화면(요소 크기·포커스 등)을 만져야 하면 mounted. 헷갈리면 mounted를 쓰면 대체로 안전합니다.

methods + async / await

⑨ 서버에서 데이터 가져오기 (비동기)

서버 응답은 시간이 걸립니다. async 함수 안에서 await로 "응답이 올 때까지 기다렸다가" 다음 줄로 넘어갑니다. 보통 mounted에서 호출합니다.

<script>
methods: {
  async loadUsers() {
    this.loading = true;             // 로딩 시작
    const res = await fetch("/api/users"); // 응답 기다림
    this.users = await res.json();   // 받은 데이터 저장
    this.loading = false;            // 로딩 끝
  }
},
mounted() {
  this.loadUsers();   // 화면 켜지면 자동 호출
}
실제 동작 (응답 지연을 흉내냅니다)
⏳ 응답을 기다리는 중...
  • {{ u.name }}
버튼을 눌러 데이터를 불러오세요.

await로 기다리는 동안 loading이 true라 "불러오는 중"이 보이고, 응답이 도착하면 목록으로 바뀝니다. 실제로는 fetch나 axios로 서버를 호출합니다.
