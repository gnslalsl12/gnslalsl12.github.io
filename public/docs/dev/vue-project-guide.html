화면이 뜨기까지 무슨 일이 일어날까요

Vue가 처음이라도 괜찮습니다. 이 프로젝트는 거의 모든 화면이 똑같은 한 가지 흐름으로 동작합니다. 그 흐름만 손에 익히면, 어떤 화면을 고치든 새로 만들든 길을 잃지 않습니다.

한눈에 보는 큰 그림

이 프로젝트의 거의 모든 화면은 이렇게 동작합니다

아래 흐름을 외워두면 절반은 끝납니다. 화면을 고칠 때도, 만들 때도 매번 이 순서를 거꾸로 혹은 정방향으로 따라가게 됩니다.

rendering-flow.txt 
브라우저에서 URL이 정해짐
        ↓
router/index.js   — 이 URL이 어떤 컴포넌트인지 사전(辭典)에서 찾음
        ↓
   해당 Route 매칭
        ↓
   Index.vue   — 그 영역의 "틀(레이아웃)"이 되는 상위 페이지
        ↓
<router-view />   — 자식 화면이 들어올 "빈자리"
        ↓
   실제 화면.vue   — 우리가 보통 실제로 고치는 파일
▶

핵심 한 줄. URL은 스스로 화면을 알지 못합니다. router/index.js라는 "주소록"이 "이 URL이면 이 컴포넌트"라고 일일이 연결해 줍니다. 이게 Next.js와 가장 다른 점입니다.

지도부터 펴기

폴더 구조 — 어디에 뭐가 있나

전부 외울 필요는 없습니다. 실제로 자주 여는 폴더는 router, views, menu 세 군데입니다.

src
├── api          # 서버와 통신하는 함수 모음
├── assets       # 이미지·폰트 등 정적 파일
├── components   # 여러 화면이 공유하는 작은 부품
├── router
│   └── index.js         # ★★★ URL ↔ 컴포넌트 연결 주소록
├── store        # 전역 상태(여러 화면이 공유하는 데이터)
├── utils        # 공통 유틸 함수
├── views        # ★★★ 화면(페이지) 단위 .vue 파일들
│   ├── Template      # 메뉴 한 덩어리 = 폴더 하나
│   ├── Project
│   ├── OCR
│   ├── Learning ...
│   ├── Login.vue
│   └── User.vue
├── App.vue      # 최상위 껍데기. 현재 URL의 화면을 출력
└── main.js      # 앱 시작점
▶

views 안에서 메뉴 하나가 폴더 하나로 묶이는 경우가 많습니다. 예) 템플릿 관련 화면들은 전부 views/Template/ 안에 모여 있습니다.

⚠

중요. 좌측 메뉴는 더 이상 adminMenu.json으로 만들지 않습니다. 지금은 DB의 ocr_menu 테이블에 메뉴 이름과 링크(URL)를 넣어두고, 그 링크가 라우트와 매칭되면 연결되는 방식입니다. 그래서 메뉴를 손볼 땐 파일이 아니라 ocr_menu 테이블을 봐야 합니다.

헷갈리는 지점 먼저 정리

Next.js를 써봤다면, 딱 이 차이만 기억하세요

Next.js는 "폴더를 만들면 URL이 자동으로 생긴다"였지만, 이 Vue 프로젝트는 "내가 직접 주소록에 등록해야 URL이 생긴다"입니다.

Next.js

자동 연결

파일 위치가 곧 URL입니다. 폴더만 만들면 라우팅이 따라옵니다.

app/template/page.tsx
        ↓  (자동)
   /template
VS
이 Vue 프로젝트

수동 연결

파일을 만들어도 자동 연결되지 않습니다. router/index.js에 직접 등록해야 URL이 생깁니다.

views/Template/Group.vue
        +
router/index.js 에 등록
        ↓  (수동)
   /template
⚠

가장 흔한 실수. .vue 파일만 만들고 "왜 화면이 안 떠?" 하는 경우 — 십중팔구 router/index.js 등록을 빠뜨린 것입니다.

등장인물 소개

자주 만나게 될 핵심 파일 4가지

각각이 무슨 일을 하는지만 알면 됩니다. 카드를 눌러 더 자세히 볼 수 있습니다.

실전 1 · 고칠 파일 찾기

"이 화면, 대체 어느 파일이지?" 추적하기

화면은 보이는데 어떤 파일인지 모를 때, 메뉴 이름에서 출발해 실제 파일까지 따라가는 7단계입니다. 다음을 눌러 한 단계씩 따라와 보세요.

출발 · 메뉴

화면에 보이는 메뉴 이름을 확인

예시로 좌측 메뉴 "템플릿 관리"를 고치고 싶다고 합시다.

화면에서 본 메뉴
템플릿 관리
메뉴 → URL

ocr_menu 테이블에서 그 메뉴의 link(URL) 확인

DB의 ocr_menu 테이블에서 메뉴 이름으로 조회해 어떤 URL로 가는지 알아냅니다. (컬럼명은 프로젝트 스키마에 맞게)

ocr_menu 테이블 조회
SELECT link FROM ocr_menu
WHERE menu_name = '템플릿 관리';
-- 결과: /template
URL → Route

router/index.js에서 그 URL(path)을 검색

path: "/template"로 검색하면 해당 Route 블록이 나옵니다.

src/router/index.js
{
  path: "/template",
  component: Template,   // ← 이 컴포넌트가 상위 페이지
  children: [ ... ]
}
Route → 파일 위치

그 component가 어디서 import 됐는지 확인

파일 위 import 구문을 보면 실제 파일 경로를 알 수 있습니다.

src/router/index.js (상단)
import Template from "@/views/Template/Index.vue";
상위 페이지 열기

Index.vue를 열어 <router-view>가 있는지 본다

자리만 비어 있다는 건, 진짜 내용은 자식(children) 화면에 있다는 신호입니다.

views/Template/Index.vue
<template>
  <Layout>
    <Title />
    <router-view />   <!-- 자식 화면이 여기로 -->
  </Layout>
</template>
children 확인

router의 children에서 기본 화면을 찾는다

path: "" (빈 문자열)이 /template의 기본 화면입니다.

src/router/index.js (children)
children: [
  { path: "", component: TemplateGroup },  // ← 기본 화면
  { path: "group/add", component: GroupAdd },
  { path: "add", component: TemplateAdd }
]
도착 · 고칠 파일

그 component의 실제 .vue 파일이 정답

결국 우리가 손대야 할 파일은 자식 컴포넌트입니다.

고치면 되는 파일
src/views/Template/Group.vue   ✓
▶

요약. 메뉴 이름 → link(URL) → router의 path → component → import 경로 → Index.vue의 router-view → children의 실제 파일. 이 사슬을 거꾸로 타고 내려오면 됩니다.

실전 2 · 화면 새로 만들기

새 페이지 추가하기 — "템플릿 복사" 화면 예시

순서가 핵심입니다. ① 파일을 만들고 → ② 주소록(router)에 등록하면 URL이 살아납니다. 메뉴 등록은 선택입니다.

.vue 파일 생성

화면 폴더 안에 새 파일을 만듭니다. 위치는 같은 메뉴 그룹 폴더 안이 자연스럽습니다.

src/views/Template/TemplateCopy.vue
<template>
  <div>템플릿 복사 화면</div>
</template>

<script>
export default {
  name: "TemplateCopy",
};
</script>

router/index.js에 등록

해당 영역의 children 배열에 한 줄 추가합니다. import(...) 방식은 "필요할 때 불러오기"라 무거운 화면에 좋습니다.

src/router/index.js
{
  path: "copy",
  name: "TemplateCopy",
  component: () => import("@/views/Template/TemplateCopy.vue"),
}

최종 Route 모습 확인

기존 children에 copy가 자연스럽게 끼어든 형태가 됩니다.

완성된 children
children: [
  { path: "",          component: TemplateGroup },
  { path: "group/add", component: GroupAdd },
  { path: "add",       component: TemplateAdd },
  { path: "copy",      component: () => import("@/views/Template/TemplateCopy.vue") },
]

접속 URL 확인

상위 /template + 자식 copy = 최종 URL. 부모 path와 자식 path가 합쳐집니다.

브라우저 주소
/template/copy

메뉴에 노출 (선택)

좌측 메뉴에서 바로 들어가게 하려면 DB의 ocr_menu 테이블에 행을 추가합니다. link가 방금 만든 URL과 같으면 메뉴 → 화면이 연결됩니다. 직접 URL로 접속한다면 생략해도 됩니다.

ocr_menu 테이블에 행 추가
INSERT INTO ocr_menu (menu_name, link)
VALUES ('템플릿 복사', '/template/copy');
가장 궁금했던 부분

.vue 파일은 어떻게 생겼나 — 3칸 구조

하나의 .vue 파일은 세 칸으로 나뉩니다. 화면 모양(template), 동작·데이터(script), 꾸밈(style). 이 셋이 한 파일에 모여 "한 화면"을 이룹니다.

<template>
화면 모양 (HTML)
눈에 보이는 화면을 그리는 칸입니다. 일반 HTML에 Vue 문법({{ }}, @click, v-if 등)을 섞어 씁니다. 
<div>
  <h2>{{ title }}</h2>        <!-- 데이터 표시 -->
  <button @click="save">저장</button>  <!-- 클릭 시 save 실행 -->
</div>
<script>
동작 · 데이터 (JS)
이 화면의 "두뇌"입니다. export default { ... } 안에 데이터와 함수를 정해진 이름의 칸(옵션)에 나눠 담습니다. 아래에서 칸마다 자세히 봅니다. 
<script>
export default {
  name: "TemplateCopy",
  data() { return { title: "템플릿 복사" }; },
  methods: { save() { /* 저장 로직 */ } },
};
</script>
<style>
꾸밈 (CSS)
이 화면의 디자인을 입히는 칸입니다. scoped를 붙이면 이 파일 안에서만 적용돼, 다른 화면 디자인을 건드리지 않습니다. 
<style scoped>
h2 { color: #2E9E6E; }   /* 이 화면의 h2에만 적용 */
</style>

그래서 <script> 안에는 뭘 적나요? — 옵션(칸) 사전

이 프로젝트는 Options API 방식입니다. 즉 정해진 이름의 칸에 내용을 나눠 담습니다. 칸을 눌러 펼쳐 보세요.

이 컴포넌트의 이름입니다. 개발자 도구나 에러 메시지에서 화면을 알아보기 쉽게 해줍니다. 보통 파일명과 같게 짓습니다. 동작에 큰 영향은 없지만 적어두는 습관이 좋습니다.

name: "TemplateCopy",

다른 .vue 컴포넌트(예: 공통 버튼, 모달, 표)를 이 화면에서 쓰려면 먼저 import 하고 여기에 등록해야 <template>에서 태그처럼 쓸 수 있습니다.

import Modal from "@/components/Modal.vue";
// ...
components: { Modal },   // 이후 <Modal /> 로 사용 가능

바깥에서 전달받는 값입니다. 부모 컴포넌트가 넘겨주거나, router에서 props: true로 설정하면 URL 파라미터가 props로 들어옵니다. 받은 값은 <template>에서 바로 쓸 수 있습니다.

props: {
  id: String,   // /detail/123 → id 가 "123"
},

화면 안에서 바뀌는 값들을 담는 곳입니다. 입력창 내용, 목록 데이터, 로딩 여부 등. 반드시 함수로 만들어 객체를 return 합니다. 여기 담긴 값이 바뀌면 화면이 자동으로 다시 그려집니다.

data() {
  return {
    keyword: "",      // 검색어 입력값
    list: [],         // 화면에 뿌릴 목록
    isLoading: false,
  };
},

다른 값들을 조합해 자동으로 계산되는 값입니다. 재료가 되는 값이 바뀌면 알아서 다시 계산됩니다. "필터링된 목록", "합계" 같은 데 씁니다. 첨부 문서의 setTitle처럼 화면 제목을 상황에 따라 바꾸는 데도 좋습니다.

computed: {
  title() {
    return this.$route.query.tpType === "R"
      ? "인식 템플릿 관리"
      : "분류 템플릿 관리";
  },
},

어떤 값이 바뀌는 순간 무언가를 실행하고 싶을 때 씁니다. 예) 검색어가 바뀌면 서버에 다시 요청. (계산된 결과가 필요하면 computed, "바뀌면 ~해라"가 필요하면 watch)

watch: {
  keyword(newVal) {
    this.search(newVal);   // 검색어가 바뀌면 재검색
  },
},

이 화면의 동작(함수)을 모아두는 곳입니다. 버튼 클릭, 저장, 페이지 이동 등. <template>에서 @click="save"처럼 연결해 호출합니다. 안에서는 this.데이터로 data 값에 접근합니다.

methods: {
  save() { /* 저장 처리 */ },
  moveCopy() {
    this.$router.push("/template/copy");  // 페이지 이동
  },
},

화면이 만들어지는 특정 시점에 자동으로 실행되는 칸입니다(생명주기 훅). 보통 화면이 열리자마자 서버에서 데이터를 가져올 때 created나 mounted를 씁니다.

created: 데이터·로직 준비가 끝난 직후 (화면 그려지기 전). mounted: 화면이 실제 화면에 붙은 직후 (DOM 사용 가능).

created() {
  this.fetchList();   // 화면 열리자마자 목록 불러오기
},
▶

순서 팁. 보통 위에서부터 name → components → props → data → computed → watch → created/mounted → methods 순으로 적습니다. 정해진 규칙은 아니지만, 팀 코드가 이 순서면 맞춰주는 게 읽기 좋습니다.

자주 쓰는 동작

페이지 이동 · 값 전달하기

버튼을 눌러 다른 화면으로 보내거나, 다음 화면에 값을 넘겨주는 두 가지 방법입니다.

① 버튼으로 페이지 이동

이동시킬 화면.vue
<template>
  <button @click="moveCopy">복사 화면으로 이동</button>
</template>

<script>
export default {
  methods: {
    moveCopy() {
      this.$router.push("/template/copy");
    },
  },
};
</script>

② 값 넘기는 2가지 방법 — Path 파라미터 vs Query String

Path 파라미터

/detail/123

"무엇을 볼지" 대상을 정할 때. router에 :id로 자리를 만들고 props:true를 켜면 props로 받습니다.

// router
{ path: "detail/:id", props: true }

// 컴포넌트
props: { id: String }
// id === "123"
또는
Query String

/template?tpType=R

"어떻게 볼지" 옵션을 줄 때(필터·모드 등). this.$route.query로 읽습니다. 첨부 문서의 제목 분기와 동일합니다.

// 컴포넌트
this.$route.query.tpType
// "R" → 인식 템플릿 관리
// "C" → 분류 템플릿 관리
▶

구분 요령. URL 경로에 박히는 "그 대상"이면 path 파라미터(/detail/123), 화면을 보는 "옵션/모드"면 query(?tpType=R)를 씁니다.
